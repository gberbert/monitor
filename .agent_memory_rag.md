### [2025-12-27 21:40:00] - Implementação Final: NVR Nativo (Go2RTC + Python Recorder)
- **Problema Crítico**: A tentativa de usar **Frigate NVR via Docker no Windows** falhou devido a limitações severas de conectividade de rede (Firewall Host vs Docker Isolation) que impediam o container de acessar as câmeras IP ou o host de acessar o container.
- **Solução (Pivô)**: Abandonada a virtualização/containerização para o NVR. Implementada uma arquitetura 100% nativa no Windows.
- **Arquitetura Implementada**:
  - **Core de Conectividade**: `Go2RTC v1.9.13` (Nativo) ouvindo na porta 8554 (RTSP) e 1984 (API). Responsável por manter a conexão estável com as câmeras.
  - **Motor de Gravação**: `recorder.py` (Python). Conecta ao `localhost:8554` (Go2RTC) e spawn processos `ffmpeg` persistentes para gravar segmentos de 60s.
    - **Codec**: Transcodificação forçada para **H.264/AAC** (`-c:v libx264 -preset ultrafast -c:a aac`) para garantir reprodução em navegadores sem suporte a H.265.
  - **Indexação**: `indexer.py` (Python). Varre a pasta `go2rtc_bin/storage` periodicamente, valida arquivos com `ffmpeg` (duração real) e popula `monitor.db` (SQLite).
  - **Backend**: `nvr_api_new.py` (Flask). Serve os metadados para o frontend.
  - **Frontend**: `timeline_demo.html`. Interface visual para navegação.
  - **Orquestração**: `start_nvr_native.bat`. Script único que inicia todos os componentes em paralelo.
- **Status**: **SUCESSO**. O sistema grava, indexa e reproduz vídeos de forma estável, contornando todos os problemas de rede anteriores.
